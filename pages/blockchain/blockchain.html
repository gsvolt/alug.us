So What is Blockchain?

It's basically Git

Here are the similaries between the two.

Git Blockchain
Linked List, history of commits Linked list, chain of blocks
Each commit has a hash of its contents Each block has a hash of its contents
Commit points to parent Block points to parent
History is append only Chain is append only
Use merkle tree to store blobs (files / data) Use merkle tree to store transactions
Distributed to anyone who clones it Distributed to anyone that runes the node software
Verifed using commit hashes Verified using block hashes

However there are some key important differences.

Git Blockchain
Anyone can rewrite history No one can rewrite history
No global agreement needed Global consensus mechanism (proof of work (PoW), proof of stake (PoS), etc)
No economic incentive Economic incentives for mining, or having a node
Rely's on trust and who you add as a collaborator Trust only the code and consensus mechanism

Ok but how does it work?

Blockadelphia

Basically there's a few steps to any blockchain, and they all work with the same underlying principals and use the same general logic loop.

  1. Submit transaction
  2. Transaction gets added to "mempool"
  3. Transaction limit or time limit gets reached (arbitrarily set by chain)
  4. Miners race to mine the block using a verification algorithm (PoW and PoS have many different implementations)
  5. One node wins, block is added to chain, process repeats

What's the problem with Proof of Work? (i.e. Bitcoin)

Problems Schmablems

  1. Crazy waste of Energy
  2. Incredibly inefficient as "Money", transaction throughput is really bad.
  3. Not really a store of value like its claims of "Digital Gold"
  4. Very user unfriendly

What about this new fangled Proof of Stake? (i.e. Ethereum)

I heard it's all the rage

  1. Relies on people having a lot of money to run nodes.
  2. Genesis creates a lot of tokens for the people who started the protocol (pyramid / ponzi scheme?)
  3. Most are deflationary, I'm not an economist, but deflationary currency is a bit of an oxymoron
  4. Very user unfriendly
